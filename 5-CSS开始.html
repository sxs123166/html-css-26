2.内部样式表（嵌入样式表） 样式写到head标签中的style标签中，只能控制当前页面中的标签 3.外部样式表（链接样式表） 样式写到外部的css文件中，多个页面可以共享同一个css文件，实现样式的统一管理 选择器 1.基础选择器 1.1 类型选择器 选择器选中的是文档中所有的指定类型的标签 p{ color: red; font-size: 20px; } 1.2 类选择器 选择器选中的是文档中所有的指定类名的标签
  • .class{ color: red; font-size: 20px; } .class2 { } 注意： 1.类名自定义，不能是中文、纯数字 2.多个英文单词组成尽量使用 - 链接 3.命名要有意义，尽量见名知意 4.class属性可以有多类名中间用空格隔开 1.3 ID选择器 选择器选中的是文档中所有的指定ID名的标签 #id{ color: red; font-size: 20px; }
    类与 id的区别 类选择器： 语法：以.开头，后跟类名(如.nav) 作用：选择 class 属性的一个或多个元素 特点：可以使用多次 类似：身份证的名字，可以使用多次 场景：后期修改样式基本都是类选择器 id选择器： 语法：以#开头，后跟id名(如#header) 作用：以特定id属性的唯一元素 特点：同一页面中，id值必须唯一（不能重复） 类似：身份证的编号，唯一，只能用一次 长截屏：后期主要是配合 JavaScript添加交互效果 1.4 通配符选择器 选择器选中的是文档中所有的标签 *{ margin: 0; padding: 0; } 特殊情况下通过通配符清除所有元素的默认边距和填充，统一不同浏览器的默认样式。 * { margin: 0; /* 清除所有元素的外边距 */ padding: 0; /* 去除所有元素的内边距 */ box-sizing: border-box; /* 统一盒模型 */ } Css的层叠性 层叠性是指多个选择器选中同一个标签时，多个样式会层叠在一起，最终生效的样式是根据层叠性规则来确定的。 层叠性规则： 1. 内联样式表（行内样式表）的样式优先级最高 2. 内部样式表（嵌入样式表）的样式优先级次之 3. 外部样式表（链接样式表）的样式优先级最低 后定义的样式覆盖先前的样式 书写规则： 1.选择器和大括号中间保留一个空格距离 2.属性名和属性值中间也要保留一个空格 3.每个 CSS 属性单独占一行。后期打包无需担心体积问题 总结 选择器类型 语法示例 匹配范围 复用性 典型使用场景 注意事项 类型选择器 div{} 匹配所有指定HTML元素 某一类型标签 全局样式重置、基础标签样式 避免滥用、可能导致样式冲突 类选择器 .nav{} 匹配所有含指定 class的元素 多次使用 多元素共享样式 优先使用，避免与 ID 选择器冲突 ID 选择器 #deader{} 匹配唯一含指定 id的元素 唯一性 唯一元素标识 必须唯一，避免样式覆盖风险 通配符选择器 *{} 匹配所有元素 全局覆盖 全局样式重叠 性能较差，慎用 大部分情况下，我们使用类选择器，是最重要的 2.关系选择器 关系选择器是通过位置关系来选择目标元素（标签），可以更精准选择某些元素。常见的有： 2.1 后代选择器 选择某个元素的后代元素（不限层级，包括子元素、孙元素等） 父级和子级可以是任意选择器 .box p { color: pink; } 可以有多个层级 div p a{ color: pink; } 可以直接从子代开始： .box p { }
